# Визуализация системности

- [[визуальное представление информации]]
- [[Визуализация для освобождения ресурса мозга]]
- [[Визуализация системности не доложна быть громоздкой]]
- [[Принцип наглядности в педагогике]] coupled with my [[Теория системного обучения]]
- Ways
	- One common graph of one module of lessons (for example, psychology in the 8 grade) and detailed one for each lessons
		- Vertexes of lessons graph is a concrete ideas
			- [[Способы визуализации графа идей урока]]
		- Then we have 2 ways to make module graph:
			- Vertexes of module graph is [[Понятие]]
				- Description - [[Представление кластеров знаний уроков как вершин графа как средство систематизации знаний ученика]]
			- Vertexes of module graph is concrete ideas
			- Thoughts
				- Module graph that contain ideas instead of concepts will be more clear for students, but it might be too noisy
	- One big graph with connected ideas
		- Ideas which should be learned today mentioned with color
			- Students see which ideas they will learn today, and which not yet learned ideas are linked with them
		- Problems
			- This graph might become messy
			- If we show it to students, we will spoiler them links that had better to be done with students themselves
- На основе этого может строиться хорошее [[Домашнее задание]] или облегченное [[Тестирование в педагогике|проверочное задание]]: попросить обяснить связи между определенными понятиями/идеями на графе, рассказать по нему систему
	- Если же это дз, то попросить сначала попытаться [[Практика извлечения|самому воспроизвести систему]], затем посмотреть на граф и попытаться обяснить ее по нему
	- Похоже на идею [[Задание на описание соотношения понятий]], но тут делается акцент именно на обяснении всех связей в системе
