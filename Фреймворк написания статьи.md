# Фреймворк написания статьи
- Когда вы создаете кластер знаний по какой то теме, не нужно находить большое количество материалов по ней. Найдите одну ссылку, прочитайте ее. Если не нашли ничего полезного и никаких внутренних ссылок, открывайте следующую ссылку
	- Если вы найдете слишком много материала, это будет угнетать вас
- Для начала вам нужно создать [[Кластер знаний]]
	- Для этого вы можете применить [[Фреймворк изучения темы]]
	- Читаете статью в зотеро - экспортируете аннотации в обсидиан - переносите их в канвас, соединяете между собой, с другими карточками, с вашими мыслями, которые появляются по ходу - в процессе формулируете гипотезы, вопросы для дальнейшего поиска, выделяете их отдельным цветом
	- Когда информации накопилось достаточно или поджимают [[дедлайн]]ы, переносите все в атомарные заметки (где это необходимо, см. [[Принцип достаточной атомарности]]), соединяете их друг с другом
- Затем, исходя из темы статьи составьте план
- Расположите все имеющиеся заметки в соответствии с планом
- Переведите каждую заметку в цельный текст в отдельном файле
- Соедините все пункты статьи в одном файле
- У читателя должна сложиться система связей по вашей теме
	- Которая будет состоять как из ссылок на внешние темы, так и ссылки одного блока статьи на другой
- Важное правило - [[У каждого пункта текста должны быть прописаны не только ссылки, но и беклинки]]
- Написание статьи по изученной теме очень хорошо структурирует ваши знания
	- А также поможет открыть новые связи в процессе написания - [[Новые связи рождаются в процессе перевода конспекта в целостный текст]]
- Если в процессе вы будете создавать свои концепции, перед написанием статьи не забудьте провести [[Ревизия понятий с помощью цеттелькастена]]
